How it works

同じ文字集合なら、いつでも同じフォント。

入力順や重複に左右されない文字集合を作り、ソースフォント・生成設定・文字集合をまとめてハッシュ化します。

01 / COLLECT

ページから文字を収集

SDK が表示対象のテキストだけを走査。非表示要素やスクリプトを除外し、DOM 更新にも追従します。

02 / NORMALIZE

安全な文字集合へ正規化

NFC 正規化、制御文字の除去、重複排除、コードポイント順の整列で安定した入力を作ります。

03 / GENERATE

隔離プロセスで生成

FontTools を独立プロセスで実行。時間・同時実行数・待機数・キャッシュ容量を制限します。

04 / VALIDATE

WOFF2 を検証

署名、ファイルサイズ、フォント構造を確認してから、原子的にキャッシュへ配置します。

05 / DISTRIBUTE

不変 URL で配信

フォント版と文字ハッシュを含む URL を返し、CDN とブラウザで一年間 immutable キャッシュします。

06 / EXTEND

差分だけを追加

MutationObserver が新しい文字を見つけたとき、既に読み込んだ文字を除外して差分を要求します。

角の静けさと、丸のやわらかさ。

Google Fonts の Zen ファミリーを、本文と見出しの役割に合わせて配信します。

zen-kaku-regular / 400

zen-maru-regular · bold / 400 · 700

やさしい輪郭に、意志を宿す。

Zen Maru Gothic — 見出し、メッセージ、印象的な言葉に。

Public surface

小さく、明示的な API。

公開面を必要最小限に保ち、生成・配信・監視を役割ごとに分離しています。

Guardrails

壊れ方まで、設計する。

Request limits

ストリームを含む本文サイズと、ユニーク文字数を制限。

Process isolation

タイムアウトした生成プロセスを終了し、一時ファイルを清掃。

Capacity control

待機列、並列数、キャッシュ総量、空き容量に上限を設定。

Observable

原文を残さず、request ID・件数・ハッシュ・時間を JSON 記録。
