プライバシーポリシー

本サービスが受け取る文字データ、生成キャッシュ、運用ログおよび外部サービスの取り扱いを説明します。

制定日：2026年7月13日最終更新：2026年7月13日Version 1.0

設計原則は「本文を運用ログに残さないこと」です。API はフォント生成のために文字列を一時的に処理しますが、アプリケーションログには原文を記録しません。

基本方針

サービス提供に必要な範囲を超えて情報を収集せず、目的を明確にし、アクセス制御・容量制限・ログ最小化などの技術的措置を講じます。本サービスはユーザーアカウント、広告プロファイルまたは行動追跡機能を提供しません。

処理する情報

フォント生成リクエスト

  • 要求するフォント ID
  • サブセットに含める文字列
  • HTTP メソッド、パス、Origin、request ID などの通信情報

文字列は正規化・重複排除された後、FontTools の隔離プロセスへ渡されます。文字列はアカウント情報と結び付けられません。

自動的に送信される情報

通信時には、IP アドレス、User-Agent、時刻、TLS・ネットワーク情報などが CDN、WAF、ホスティング基盤へ送信される場合があります。

利用目的

  • 要求された WOFF2 サブセットの生成と配信
  • 同一文字集合の重複生成防止とキャッシュ利用
  • 障害調査、性能把握、不正利用防止、セキュリティ維持
  • サービス状態の監視と容量計画

アプリケーションログ

アプリケーションは標準出力へ構造化 JSON ログを出力します。記録対象は次のとおりです。

  • request ID、メソッド、パス、状態コード、処理時間
  • フォント ID、正規化後のユニーク文字数
  • 文字集合から生成した SHA-256 ハッシュ
  • キャッシュ命中、出力サイズ、エラー種別
記録しないもの： API に送信された元の文字列および正規化後の文字列そのもの。

生成キャッシュ

生成された WOFF2 は、フォント版と文字集合ハッシュに基づく不変 URL でキャッシュされます。フォントファイルには要求された文字のグリフが含まれるため、秘密情報や公開を意図しない文字列を送信しないでください。キャッシュは容量保護の対象ですが、個別 URL の保存期間は保証しません。

Cookie とローカル保存

本サイトのフロントエンドは Cookie、localStorage、広告識別子、アクセス解析タグを使用しません。ブラウザおよび CDN は、HTTP キャッシュ仕様に基づいて CSS、JavaScript、画像、フォントを保存することがあります。

外部サービス

  • CDN・WAF： 運営者の構成によっては、配信、TLS、攻撃防御、レート制限のため通信情報を処理する場合があります。
  • 外部リンク： GitHub や Google Fonts へ移動した後は各サービスのポリシーが適用されます。

保存期間と安全管理

運用ログは障害対応・セキュリティ・容量管理に必要な期間に限定して保持し、アクセス可能な担当者を制限します。生成キャッシュは不変配信の性質上、ログとは別に保持されます。保存先、保持期間、バックアップは運用上の必要と法的要請に応じて見直します。

照会・相談

本サービスはアカウントを持たず、ハッシュから元の文字列を復元する仕組みも保持しないため、特定利用者に紐づくデータの検索や削除が技術的に困難な場合があります。個別デプロイに関する相談は、その運営者へ対象 URL、日時、request ID などを添えてご連絡ください。

ポリシーの変更

サービス仕様、法令、利用する基盤の変更に応じて本ポリシーを改定します。重要な変更では、本ページの更新日と内容を明示します。
